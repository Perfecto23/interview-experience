# 一面

> 日期：2025.11.11
>
> 面的比较不爽，又是那种死板问答的形式。直接在反问环节吐槽了。

## 一、 问题

1. 你最擅长什么？就是在前端在哪些工具或者哪些工具链平台有一些突破的？

2. 打包工具你们是用啥？

3. 你现在是在职还是离职状态？

4. 这个项目（上家公司的安全托管服务项目）我们可以大概一起交流一下吗？你可以先介绍一下这个项目，然后我们再简单问几个问题。

5. 整个工程化你们团队大概有多少人在做？然后你在里面跟你同样角色的大概有多少人？

6. 你可以大概讲一下我们平时用到前端用的一些打包工具的一些不同点，平时我们传统的用 Webpack，还有 Vite 也用 Rspack，它们这三种有什么区别？

7. 你最擅长什么语言？

8. （手撕代码）那我们就 JS 写一个过滤组件，这可以做吗？我们现在用户他需要做一个页面，页面里面会有设计的一个组件，这个组件由你来完成，需要实现一个搜索过滤的一个组件。你可以做一个，但前期前面你可以假定有一些常量之类的，然后要把它去搜索，匹配到了就有结果，没匹配到就是没匹配到了。当然你也可以考虑一些防抖，就是这个需求。

9. （针对手撕代码）除了你目前考虑的这个，这考虑的这些功方法或功能之外，你没考虑其他的，例如搜索的时候它可能会出现什么情况？大小写，还有一些其他的一些场景。

10. 你介绍一下，你在 DevOps 还有 CI/CD 这一块之前做过的一些内容吧。

11. 那 AI review 这个你具体是怎么实现的呢？

12. 后面你的意思是就是通过提示词做了一次单次的大模型调用，是吧？

13. 那你的提示词是怎么写？怎么去调优呢？可能有一些那个自己的一些规则，你怎么注入进来？

14. 那你们调用大模型有用到什么包或者第三方库吗？

15. 那除此之外，你有做过哪些跟 AI 相关的应用？

16. 你除了 OpenAI 之外，有用过哪些跟 AI 相关的一些第三方包吗？

17. （追问）就是有一些针对前端的一些应用开发的一些脚手架，还有一些库，这些我不知道你有（没有接触过）？

18. 那你有做过一些后端的一些开发吗？

19. 具体是实现了一个什么样的一个能力？

20. 那你对页面性能（有没有遇到一些问题），你是怎么解决的？或者有不同的工具和方法去解决。

21. 你们之前离职的原因大概是怎样子的？

22. 你刚才提到那个插件，如果让你来独立开发成独立开发一个 IDE 的插件，你会去怎么做？在哪方面入手？

23. 好的，你有了解或者自己做过 MCP 和智能体相关的东西吗？

24. 你说的智能体是基于 Dify 自己去搭还是说从0去手写的那种？

25. 那做这个的时候有用到哪些第三方的库吗？还是说完全是自己手搓呢？


## 二、反问

1. 我觉得这场面试有点死板，就是我更希望说在面试过程中我们能探多探讨一些看法。比如说你提问我一个场景，那可能我公司做的和你们公司做的是不一样的，我们面对不同业务需求的时候有不同的考虑和取舍，我希望多一些这种面试的交流，而不是说你们问一个问题我来回答，这样一问一答我觉得没什么意思。

> 好的，建议我们收到了，后面的话我们也看一下，就是除了你们答之外，我们也做一个交流，因为我们本意也是想考察一下，就是因为说实话大家都参差不齐嘛。那问的问题，如果，所以为什么我们有像 Facebook 一样，人家是有在线编码环节的，因为我们无法肯定人家答的八股文，每个人每个候选人差不多都能达到80%，我就是我们 HR 筛选出来的简历，我们无法肯定他有没有真正胜任能力。嗯，同时我们也害怕我们自己的项目会有出现一些 delay，你这个你能理解，所以我们一般一问一答，你答得差不多了，跟我们的预算差不多，就没跟你再继续就一个话题深入交流了，我们可以每个点都得问一下哈，这个你理解一下。
