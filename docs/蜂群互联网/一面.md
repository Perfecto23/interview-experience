# 一面

> 日期：2025.11.11

## 一、 问题

1. 您最近做的一个项目是哪个？是有PC端和小程序两端，是吧？是以H5的形式嵌进去的吗？

2. 我看你写有主导一个前端架构的升级和工程化的一个体系建设，你能跟我描述一下你具体是做了哪一些吗？

3. 你们就是像模块联邦，你各自应用之间像一些CSS和JS，你怎么去做一些隔离的操作？

4. 有没有遇到过？就是可能子应用A跟子应用B同时依赖某一个 Node module 包，然后他们版本不一致的话，这种情况怎么解决？

5. 你推动那个构建工具的升级是出于什么考量呢？然后以及你比如说从 Webpack 4 升级到那个 Webpack 5，它解决了什么样痛点？

6. 然后后面又是有去向那个 Rspack 去做迁移，是吧？我看你有描述他本地编译速度提升了80%，你这一块是具体怎么量化的呢？或者说你具体是做了哪些去提升他的那种编译速度？

7. 我看你有写热爱开源和造轮子，有输出过哪些轮子？


## 二、反问

1. 主要是想了解一下业务相关的业务和产品使用的技术栈是什么？

2. 你们用的是 Taro？

3. 团队规模多大？就是有哪些人构成，就是哪些角色构成。

4. 如果我入职了会负责哪块业务啊？主要开发什么功能呢？

5. 你们公司的项目开发流程是什么样的？

6. 公司在哪里上班？
