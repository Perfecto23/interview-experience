1. 为何考虑长沙的机会？

5. **连续追问**：“简历上面写千行 bug 率低于 0.6% ，这个怎么解读？代码行数很好刷的，别人不认可你这个数据怎么办？很多人可以通过各种手段把需求多写几行代码（一行能解决的事情写5行），怎么办？”
6. **连续追问**：“假如你是组长，产品经理提了很多倒排期的需求，你觉得做完，超出了团队的交付能力，怎么办？假设你找不到人呢？那如果你不认可这个产品经理提的需求，那你是小组长会怎么做？产品坚持不能改和删需求，主管也不协调人支援，把这个事情委托给你了，让你全权负责，怎么办？团队底下的人也不理解认可需求，怎么办？”
7. **连续追问**：“有一个词叫做 owner 意识，你怎么解读一下？做了一个需求，把代码写完了之后，我们还有没有别的事情可以做了？除了承接技术债务消除 ，这个需求你还可以做什么？除了维护文档和写好注释 ，什么都做不了吗？”
8. **连续追问**：“代码门禁（静态检查）是做了一些什么样的事情？i18n YAML 文件不能写空字符串是怎么实现的？用 include 去判断，那这种有什么不足之处吗？如果要检测内网的一些链接不希望对外发布出去（切换场景）如果有很多 console.log，安排一个同学把这个 console 语句都删除掉，你有什么办法吗？不要用 Webpack 的 drop_console，你为什么要用 ‘AST处理’ 这种办法？直接用刚才的字符串匹配不就搞定了吗？同事确实把所有的 console.log 都删除掉了，我们在 code review 的时候有什么风险是需要关注的吗？”
9. **连续追问**：“昨天晚上下班，页面还能打开，第二天早上一来，打开不了了，是一片空白。可能是什么原因？如果他都什么日志和报错都没有，就是打不开怎么办？你的电脑不行，他的电脑行。那原因是什么？”
