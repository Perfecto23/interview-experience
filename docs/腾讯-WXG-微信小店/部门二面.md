# 部门二面

> 日期：2025.11.04

## 一、 问题

1. 你找一个你觉得比较有代表性的，比如说有技术难度的，或者能够体现出亮点的，你负责的部分来说一说。

2. 那我先打断一下，这个项目的背景是怎么样呢？就是你为什么要做这个事情？为什么要切过来？

3. 那我想问这里最开始是因为你感觉的特别慢，对吧？那么有没有分析过里头究竟慢在什么环节？

4. 另一个问题就是你怎么确保说你在你的更换过程中你的实现一定是对的，就是你构建产物就一定是符合预期的？

5. 我刚才想问你有没有什么渐进式的办法？所以你这里是等于切完构建之后就直接切构建，但是产物的话就灰度上线，是这个意思吗？

6. 有没有一些业务层面的难点或亮点？

7. 然后这些是都，比如说所有的品牌都会打到同一个包里面？

8. 那最后面你提供给这个客户的时候，他是不会包含其他客户的这部分的代码的吗？

9. （追问）那假设我们就想做得比较干净纯粹，有没有技术手段去规避？就是别让它出现在它的代码包里面。

10. 有没有过服务端？比如说用 Node.js 来写的经验呢？

11. 小程序方面有没有接触？ C 端方面的经验吗？

12. 那么你的前后端是怎么结合的呢？如果这个 html 文件有更新的话，你们是怎么更新的？

13. （追问）你们有没有什么这方面的变更流程呢？


## 二、反问

1. 我就是想问问你刚刚这个问题的解答，就是说你们是怎么做这种版本发布，然后这种 index 或者说 js 更新的问题？

2. 我还有一个问题，就是增哪些场景会用到增量构建？还有就是怎么做这个增量构建？

3. 小程序它有一个限制，就是好像是每一个包不能大于2M。像腾讯还有微信，它你们内部开发小程序的时候，假设这个它的功能或者说业务比较复杂，真的能维持在这个限制以下，就是不超出吗？

4. 你们内部的微前端的那种最佳实践是怎么样选择一个技术方案？

5. 业务相关的，可能想问问我这个岗位入职以后会做什么？

6. 对于今天的这场二面有没有什么对我的评价和建议啊？
